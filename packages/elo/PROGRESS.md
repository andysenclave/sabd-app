## Notes / open questions

- **Test runner:** node:test per the brief's nudge. Node v24.3.0 runs `.ts` test files
  natively via type stripping; `tsconfig.json` sets `erasableSyntaxOnly` to guarantee
  the source stays strippable (no enums/namespaces). Dev deps: typescript, @types/node.
- **`applyResult(playerState, result)`:** per the §5 signature, `playerState`
  (`{ rating, gamesPlayed }`) is authoritative; the `playerRating`/`gamesPlayed` copies
  inside `RoundResult` are contract fields carried for the game loop and are not read
  by the math.
- **Rating floor value:** the brief lists "rating floor" as a config constant but gives
  no value — chose **100**. Trivially tunable in `src/config.ts`.
- **`updateWordRating` gating:** "off by default" implemented as a config flag
  (`wordRatingUpdatesEnabled: false`); when disabled the function returns the word
  rating unchanged with `applied: false`. Pass a config with the flag true to activate
  (Phase 2). Return shape (`{ newWordRating, delta, expected, kFactor, applied }`) was
  unspecified in the brief.
- **Timeout breakdown:** for unsolved rounds the breakdown is all zeros
  (`{ solveBase: 0, speedBonus: 0, hintPenalty: 0 }`) since no components apply.
- **Defensive clamps:** `T` is clamped to [0,1] (overtime can't produce a negative
  speed bonus); duplicate entries in `hintsUsed` are deduped and capped at 2;
  `timeLimitSec <= 0` on a solved round throws `RangeError`.
- **Rounding:** JS `Math.round` (half-up toward +∞) used literally per the spec's
  `round(...)`.
- With default weights the 0.05 solve floor never binds (worst solve = 0.10); it only
  matters under retuned weights — covered by a config-override test.
- Git intentionally NOT initialized (user decides on version control).
